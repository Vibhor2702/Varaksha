02
Privacy Layer — Pseudonymization
HMAC Pseudonymization + Rotating Key Vault
Before any agent ever sees a UPI ID, the Rust gateway replaces all identifiers with deterministic HMAC-SHA256 pseudonyms. The mapping is held in an ephemeral session context only — never persisted to disk. Keys rotate every 15 minutes. A compromised key exposes one 15-minute window only.
// Rust — called via PyO3 binding
fn pseudonymize(upi_id: &str, session_key: &[u8]) -> String {
  let mac = HmacSha256::new_from_slice(session_key)?;
  mac.update(upi_id.as_bytes());
  hex::encode(mac.finalize().into_bytes())
}
// Real UPI ID never leaves this function

Tier 0 — Cost Optimization
Rule-Based Fast-Path Triage (85% of Traffic)
At 7,500 TPS, running every transaction through the full ML pipeline is expensive. Production introduces a Tier 0 deterministic rule engine that handles the clear cases instantly: amount below ₹500 + known merchant + no velocity anomaly = ALLOW immediately. Amount above ₹2L + new account + known mule receiver = BLOCK immediately. Only the ~15% ambiguous transactions ever reach the ML pipeline. This cuts TEE compute costs by ~85%.
COST IMPACT — Reduces TEE node requirement from ~200 nodes to ~30 nodes at peak UPI load. Infrastructure cost drops from ~₹4.2Cr/month to ~₹0.6Cr/month at scale.

Agent Behavior Monitoring
Agents that guard themselves
Varaksha doesn't just secure data flowing between agents. It watches the agents themselves. This is what prevents a compromised or jailbroken agent from producing systematically biased outputs that slip through the data-level checks.
WHAT CAN GO WRONG WITH THE AGENTS THEMSELVES
!
Threat Class: Agent Drift
Gradual Output Manipulation via Accumulated Prompt Injection
An attacker doesn't need to inject one large payload. They can send thousands of legitimate-looking transactions that each nudge the LLM agent's behavior slightly, gradually shifting its output distribution toward systematically underscoring fraud risk. This is invisible to per-message checks but visible in aggregate output statistics over time.
SLOW POISONINGINVISIBLE PER-TXVISIBLE IN AGGREGATE
!
Threat Class: Token Bloat
Jailbreak Attempts via Oversized Context
Some jailbreak techniques work by flooding the agent's context window with repeated text designed to override system instructions. The agent starts consuming 10x normal tokens per request. This is both a security signal (possible jailbreak attempt) and a DoS vector (cost explosion). The monitor catches it via token-per-request Z-score.

RESPONSE PROTOCOL — WHEN AN AGENT MISBEHAVES
L1
Z-Score 2.0–3.0
WATCH — Elevated flag, continue processing
Output distribution is shifting but within operational bounds. Dashboard shows a yellow indicator. Incident ticket created automatically. Processing continues. Monitor increases sampling rate for this agent from 100% to continuous for the next 500 transactions.
L2
Z-Score 3.0–5.0
ALERT — Flag raised, human review triggered
Significant deviation. Dashboard shows red. Automated alert fires to security team. Transactions processed by this agent in the last 100 calls are flagged for human review in the audit queue. Agent continues processing but every output is logged with a DEGRADED tag and confidence is penalized by 0.2.
L3
Z-Score > 5.0
ISOLATE — Agent removed from pipeline, fallback activated
Critical deviation. Agent is immediately removed from the LangGraph routing. A fallback model (pre-configured backup, simpler and more conservative) is substituted. All in-flight requests to the isolated agent are terminated. Full incident report written. The isolated agent's recent output history is preserved for forensic analysis. Human approval required to restore.
