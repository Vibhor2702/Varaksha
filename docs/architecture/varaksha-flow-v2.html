02
Privacy Layer · Rust Core
HMAC Pseudonymization + 15-Minute Rotating Key Vault
Before any agent sees a UPI ID, the Rust gateway replaces all identifiers with deterministic HMAC-SHA256 pseudonyms. Key mapping is held in ephemeral session memory — never persisted to disk. Keys rotate every 15 minutes. A leaked key exposes only a 15-minute window of pseudonyms, not the underlying IDs.
fn pseudonymize(upi_id: &str, session_key: &[u8]) -> String {
  let mut mac = HmacSha256::new_from_slice(session_key)
    .expect("invalid key length");
  mac.update(upi_id.as_bytes());
  hex::encode(mac.finalize().into_bytes())
  // real UPI ID never leaves this function
}

VERIFIER DESIGN — WHY LIMITED CONTEXT IS INTENTIONAL
V1
Verifier Architecture
Small Context. Single Purpose. Cannot Be Reasoned Out Of.
The verifier is deliberately not a general-purpose AI. It receives a narrow, structured prompt with only four elements: the stated objective, the specific code delta, the justification text, and the tool call type. It does not receive the full codebase, the conversation history, or any other context. This is not a limitation — it is the security property. A verifier with full context can be reasoned or jailbroken into approving anything. A stateless, single-purpose verifier cannot.

This catches cases where the executor itself is compromised or manipulated.
