Prompt Injection
Adversarial strings embedded in transaction data override agent instructions. "Ignore previous analysis" passes cleanly through unprotected pipelines. Zero existing systems detect this at ingestion.
🕵️
Agent Impersonation
Multi-agent pipelines have no identity layer. A rogue process claiming to be the risk engine can inject fabricated confidence scores. Downstream agents comply without verification — by design.
📉
Gradient Drift Attack
Thousands of legitimate-looking transactions slowly poison the fraud model's output distribution. Invisible per-transaction. Detectable only in statistical aggregate over time — if you're watching.

07 / 10 — TOOL KERNEL
New: Secure Execution Kernel
Agents that cannot
authorise their own
side effects.
The Tool Kernel sits between every agent and the external world. An agent cannot write a file, call an API, or modify state without the kernel classifying the change and routing it through a verifier. The verifier has deliberately limited context — it sees only the objective, the delta, and the justification. Not the full codebase. Not the full conversation history.
🔒
Planner → Kernel
The planner emits ToolRequest(tool, args, justification). It never executes anything directly. Reads/pools pass through immediately. Every write is intercepted.
⚖️
Δ Classification
The kernel classifies writes as REFACTOR (same interface), EXTEND (new surface), or RESTRUCTURE (signature change). Higher risk class → stricter verifier threshold.
✓
Verifier Response
Returns APPROVE or REJECT(reason). Rejection injects reason back to planner. Syscall shadow log records every approved write for forensic verification.
