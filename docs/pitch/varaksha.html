fn pseudonymize(upi_id: &str, session_key: &[u8]) -> String {
    let mut mac = HmacSha256::new_from_slice(session_key)
        .expect("invalid key length");
    mac.update(upi_id.as_bytes());
    hex::encode(mac.finalize().into_bytes())
    // real UPI ID never leaves this function. crate: hmac + sha2
}

6 Technical Differentiators

THE MOAT
IS TECHNICAL.

01
Rust-Native Ed25519 Gate System
Every inter-agent message is cryptographically signed and verified. Eliminates the #1 unaddressed attack vector in AI pipelines: agent impersonation via score injection mid-pipeline.
ed25519-dalek · 0.3ms per gate · 13.7× Python speedup
02
HMAC Pseudonymization + Key Rotation
Real UPI IDs are replaced with deterministic HMAC-SHA256 pseudonyms before any AI model interaction. 15-minute key rotation — a leaked key exposes only one window of data.
hmac + sha2 crates · 15-min rotation · DPDP §8 compliant
03
SGX Hardware Enclave Graph Analysis
Mule network detection runs inside Intel SGX hardware-encrypted memory. The graph is constructed and destroyed inside the enclave. Only a risk score exits — the raw graph never does.
Intel SGX · gramine-sgx · EPC: 128MB
04
Differential Privacy on All Inputs
Calibrated Laplace noise (ε=1.0) on all numerical fields before agents. Mathematical guarantee: repeated queries cannot reconstruct any individual's exact transaction values.
IBM diffprivlib · ε=1.0 Laplace · <2.3% ROC-AUC loss
05
KL-Divergence Pre-Drift Monitor
Rolling 100-transaction KL-divergence detects batch poisoning campaigns before the model drifts. Fires before the exploit transaction is even sent — the only pre-drift detection mechanism in the pipeline.
KL-divergence · 100-tx window · AML.T0020 detection
06
Automated Legal Complaint in 2 Seconds
First auto-legal pipeline for UPI fraud in India. Every BLOCK verdict becomes a court-ready PDF with six Indian law citations, a tamper-evident audit chain, and automatic authority routing.
BNS · IT Act · PMLA · DPDP · RBI · CERT-In · <2s
